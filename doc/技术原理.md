# 技术原理

## 1. 三种部署拓扑

Open Design 是一个 Web 应用加本地守护进程。其分裂设计意味着同样的 UI 可以以三种形态运行：

### 拓扑 A — 完全本地（默认）

```
┌────────────────── 用户机器 ──────────────────┐
│                                              │
│   浏览器 ──► Next.js 开发服务器 (localhost:3000)  │
│                        │                      │
│                        │ http://localhost:7456 │
│                        ▼                      │
│            od 守护进程 (Node，长运行)         │
│                        │                      │
│                        ▼                      │
│            spawns: claude / codex / cursor / … │
└──────────────────────────────────────────────┘
```

一条 `pnpm tools-dev run web` 命令同时启动 Next.js 应用和守护进程。`pnpm tools-dev` 额外增加桌面外壳。零配置，无需账号。

### 拓扑 B — Web 在 Vercel + 守护进程在用户机器

```
浏览器 ──► od.yourdomain.com (Vercel)
              │
              │ ws(s):// 用户提供的 URL（如 cloudflared tunnel）
              ▼
        od 守护进程在用户笔记本上
              │
              ▼
        spawns: claude / codex / …
```

用户运行 `od daemon --expose` 打印 tunnel URL；他们将其粘贴到部署的 Web 应用的"连接守护进程"屏幕。守护进程持有密钥；Vercel 不持有任何敏感数据。

### 拓扑 C — Web 在 Vercel + 直接 API（无守护进程）

```
浏览器 ──► od.yourdomain.com (Vercel serverless)
                       │
                       ▼
              Anthropic Messages API (BYOK 存储在浏览器)
```

没有本地 CLI，没有守护进程。体验降级——无 Claude Code 技能、无文件系统产物（存储在 IndexedDB）、无 PPTX 导出。但这却是"随便试试"的路径。密钥存储在 localStorage 中，有明确警告。

三种拓扑共享同一个 Web 包；区别在于启用的传输方式不同。

## 3. 数据流

### 对话流程

1. **用户输入简报** → Web 前端
2. **发现表单**（Turn 1）→ 代理发送 `<question-form id="discovery">` → 前端渲染问卷
3. **用户填写表单** → 提交给守护进程
4. **视觉方向选择**（如无品牌）→ 代理发送第二个表单，包含 5 种策划方向
5. **实时 Todo 进度** → 代理通过 SSE 流式传输 TodoWrite 更新 → 前端实时显示
6. **产物解析** → 代理在 SSE 流中输出 `<artifact>...</artifact>` → 前端解析
7. **沙箱预览** → 产物渲染在 `<iframe srcdoc="...">` 中
8. **导出** → HTML / PDF / PPTX / ZIP / Markdown

### 提示词组合

每次发送时，应用从三层构建系统提示词：

```
BASE_SYSTEM_PROMPT（输出契约：用 <artifact> 包装，无代码围栏）
   + 活跃 design system 内容（DESIGN.md — 调色板/字体/布局）
   + 活跃 skill 内容（SKILL.md — 工作流和输出规则）
```

## 4. 关键 API 端点

### 守护进程 API

| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/chat` | POST/SSE | 流式聊天对话 |
| `/api/agents` | GET | 列出可用的代理 CLI |
| `/api/skills` | GET | 列出可用的技能 |
| `/api/design-systems` | GET | 列出可用的设计系统 |
| `/api/projects` | GET/POST | 项目 CRUD |
| `/api/projects/:id/files` | GET | 获取项目文件 |
| `/api/artifacts/save` | POST | 保存产物到磁盘 |
| `/api/artifacts/lint` | POST | 结构检查 |
| `/api/import/claude-design` | POST | 导入 Claude Design ZIP |
| `/api/templates` | GET | 保存的模板 |
| `/api/upload` | POST | 上传文件 |
| `/api/proxy/{provider}/stream` | POST/SSE | BYOK 代理（anthropic/openai/azure/google） |

### BYOK 代理协议

守护进程实现 OpenAI 兼容的代理端点，接受 provider-specific 上游 API 调用，将 SSE 规范化回 `delta/end/error` 格式：

- **允许:** loopback 本地 LLM 提供商（Ollama、LM Studio）
- **拒绝:** 非 loopback 私有地址、link-local、CGNAT、多播、保留地址、跳转目标
- **SSRF 防护:** 在守护进程边缘阻止内部 IP 访问

## 5. 存储架构

### SQLite 数据库

守护进程在 `.od/app.sqlite` 维护以下表：

- **projects** — 项目元数据（id、名称、创建时间、最后访问时间）
- **conversations** — 对话（属于项目）
- **messages** — 消息（属于对话）
- **tabs** — 打开的标签页
- **templates** — 保存的模板

### 文件系统结构

```
.od/
├── app.sqlite                 # SQLite 数据库
├── artifacts/                 # 保存的产物（按时间戳组织）
├── media-config.json          # 媒体生成 API 密钥
└── projects/<id>/            # 每个项目的工件目录
    ├── index.html             # 生成的原型
    ├── brand-spec.md          # 品牌规范（如有）
    ├── assets/               # 生成的资源
    └── references/           # 技能参考文件
```

## 6. 生命周期管理

所有本地生命周期流程通过 `pnpm tools-dev` 进入：

```bash
pnpm tools-dev                 # 守护进程 + Web + 桌面（后台）
pnpm tools-dev start web       # 守护进程 + Web（后台）
pnpm tools-dev run web         # 守护进程 + Web（前台，用于开发）
pnpm tools-dev status          # 显示运行中的 sidecar 状态
pnpm tools-dev logs            # 显示守护进程/Web/桌面日志
pnpm tools-dev inspect desktop status  # 检查桌面状态
pnpm tools-dev stop            # 停止所有托管的运行时
pnpm tools-dev check           # 状态 + 最新日志 + 常见诊断
```

端口通过 `--daemon-port` 和 `--web-port` 配置。

## 7. Sidecar IPC 协议

桌面应用通过 sidecar IPC 发现 Web URL（无需端口猜测）：

**Stamp 字段（5 个）:**
- `app` — 应用标识
- `mode` — 运行模式
- `namespace` — 命名空间
- `ipc` — IPC 通道
- `source` — 源标识

**IPC 命令:**
- `STATUS` — 获取运行时状态
- `EVAL` — 执行 JavaScript
- `SCREENSHOT` — 截图
- `CONSOLE` — 获取控制台日志
- `CLICK` — 模拟点击
- `SHUTDOWN` — 关闭

| 协议 | CLIs |
|---|---|

| `qoder-stream-json` | Qoder CLI |
| `copilot-stream-json` | GitHub Copilot CLI |
| `json-event-stream` | Codex、Gemini、OpenCode、Cursor Agent |
| `acp-json-rpc` | Devin、Hermes、Kimi、Kiro、Kilo、Mistral Vibe |
| `pi-rpc` | Pi |
