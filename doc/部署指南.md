# 部署指南

## 部署拓扑

Open Design 支持三种部署拓扑：

### 拓扑 A — 完全本地（默认）

```
浏览器 → Next.js 开发服务器 → od 守护进程 → Agent CLI
```

本地开发模式，通过 `pnpm tools-dev run web` 启动。

### 拓扑 B — Web 在 Vercel + 守护进程在用户机器

```
浏览器 → Vercel Web 应用 → cloudflared tunnel → 用户本地守护进程
```

适合远程访问本地运行守护进程的场景。

### 拓扑 C — Web 在 Vercel + 直接 API

```
浏览器 → Vercel serverless → 云端 API（Anthropic/OpenAI 等）
```

无需本地守护进程的最小化部署。

## Docker 部署

### 环境要求

- Docker Desktop
- Docker Compose v2

### 快速开始

```bash
# 克隆仓库
git clone https://github.com/nexu-io/open-design.git
cd open-design/deploy

# 启动
docker compose up -d

# 打开浏览器
# http://localhost:7456
```

### 环境配置

在 `deploy/` 目录创建 `.env` 文件：

```env
# 主机端口
OPEN_DESIGN_PORT=7456

# 容器内存限制
OPEN_DESIGN_MEM_LIMIT=384m

# 允许的 CORS 源
OPEN_DESIGN_ALLOWED_ORIGINS=https://yourdomain.com

# Docker 镜像标签
OPEN_DESIGN_IMAGE=docker.io/vanjayak/open-design:latest
```

### 常用命令

```bash
# 查看日志
docker compose logs -f

# 重启
docker compose restart

# 停止
docker compose down

# 拉取最新镜像
docker compose pull
docker compose up -d

# 删除所有本地数据
docker compose down -v
```

### 持久化存储

Docker 部署使用卷 `open_design_data`，挂载到 `/app/.od`。数据在容器重启和镜像更新后保留。

检查卷：
```bash
docker volume inspect open-design_open_design_data
```

## 本地开发部署

### 环境要求

- Node.js ~24
- pnpm 10.33.x
- 可选：一种支持的 Agent CLI

### 安装步骤

```bash
# 克隆仓库
git clone https://github.com/nexu-io/open-design.git
cd open-design

# 启用 Corepack
corepack enable
corepack pnpm --version   # 应显示 10.33.2

# 安装依赖
pnpm install

# 启动开发服务器（前台模式）
pnpm tools-dev run web
```

### 启动模式

```bash
# 守护进程 + Web + 桌面（后台）
pnpm tools-dev

# 守护进程 + Web（后台）
pnpm tools-dev start web

# 守护进程 + Web（前台，用于开发）
pnpm tools-dev run web

# 指定端口
pnpm tools-dev run web --daemon-port 17456 --web-port 17573

# 重启
pnpm tools-dev restart --daemon-port 17456 --web-port 17573
```

### 状态和日志

```bash
# 查看状态
pnpm tools-dev status

# 查看日志
pnpm tools-dev logs

# 完整诊断
pnpm tools-dev check

# 停止
pnpm tools-dev stop
```

### 构建

```bash
# 构建守护进程
pnpm --filter @open-design/daemon build

# 构建 Web
pnpm --filter @open-design/web build

# 类型检查
pnpm typecheck
```

## Vercel 部署

### Web 前端部署

1. Fork 仓库到 GitHub
2. 在 Vercel 导入项目
3. 配置环境变量：
   - `OD_DAEMON_URL` — 本地守护进程的 URL（通过 tunnel）
4. 部署

### 守护进程本地运行

```bash
# 启动守护进程并暴露
od daemon --expose

# 输出 tunnel URL
# wss://xxxx.localtunnel.com
```

用户将 tunnel URL 粘贴到 Vercel Web 应用的"连接守护进程"屏幕。

## nginx 配置

如果使用 nginx 反向代理，需要配置 SSE 路由：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:7456;

    proxy_buffering off;
    gzip off;

    proxy_read_timeout 86400s;
    proxy_send_timeout 86400s;
    proxy_http_version 1.1;
    proxy_set_header Connection "";

    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

关键设置：
- `proxy_buffering off` — 允许 SSE 流式传输
- `gzip off` — 禁用压缩（干扰 SSE）
- 长时间超时 — SSE 可能长时间保持连接

## 数据目录

### 目录位置

| 部署方式 | 数据目录 |
|---|---|
| 源码开发 | `<repo>/.od/` |
| Docker | `/app/.od/`（卷） |
| macOS 桌面 | `~/Library/Application Support/Open Design/namespaces/release-stable/data` |
| Windows 桌面 | `%APPDATA%\Open Design\namespaces\release-stable-win\data` |
| Linux 桌面 | `~/.config/Open Design/namespaces/release-stable-linux/data` |

### 环境变量

```bash
# 重定位所有运行时数据
OD_DATA_DIR=/path/to/data

# 仅重定位媒体配置
OD_MEDIA_CONFIG_DIR=/path/to/media-config
```

### 数据迁移

从源码开发迁移到桌面应用：

```bash
# 使用迁移脚本（推荐）
OD_LEGACY_DATA_DIR=/path/to/old/repo/.od /Applications/Open\ Design.app/Contents/MacOS/Open\ Design
```

## 故障排除

### better-sqlite3 ABI 不匹配

Node.js 版本变更后，重新编译：

```bash
pnpm --filter @open-design/daemon rebuild better-sqlite3
pnpm --filter @open-design/daemon exec node -e "require('better-sqlite3')"
```

### 媒体生成 OD_BIN 缺失

```bash
# 重建守护进程
pnpm --filter @open-design/daemon build

# 重启并检查
pnpm tools-dev restart --daemon-port 7457 --web-port 5175
ls -la apps/daemon/dist/cli.js
curl -s http://127.0.0.1:7457/api/health
```

### 守护进程 API /chat 返回 500

检查守护进程日志：
```bash
pnpm tools-dev logs
```

通常是 CLI 拒绝了参数。在 `apps/daemon/src/agents.ts` 的 `buildArgs` 中调整。

### Windows 特定问题

参见 `docs/windows-troubleshooting.md`。

## 安全考虑

### BYOK API 密钥安全

- 存储在本地 `media-config.json`
- 通过环境变量或 Settings 配置
- 不提交到 git

### SSRF 防护

BYOK 代理在边缘阻止：
- 非 loopback 私有地址
- link-local 地址
- CGNAT 地址
- 多播地址
- 保留地址
- 跳转目标

### 沙箱预览

产物渲染在 `srcdoc` iframe 中，隔离于主页面上下文。
