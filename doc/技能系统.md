# 技能系统

## 概述

Open Design 的技能（Skills）遵循 Claude Code 的 `SKILL.md` 约定，每个技能是一个文件夹，包含 `SKILL.md` + `assets/` + `references/`。

**31 种内置技能** 分两种模式：

- **Prototype（原型）** — 27 个技能，生成单页产物（落地页、仪表板、移动应用等）
- **Deck（演示）** — 4 个技能，生成水平滑动演示文稿

## 技能结构

每个技能文件夹遵循 Claude Code 的 SKILL.md 约定：

```
skills/<skill-name>/
├── SKILL.md           # 技能定义（必填）
├── assets/            # 资源文件（可选）
│   └── template.html  # HTML 模板
└── references/       # 参考文档（可选）
    ├── themes.md
    ├── layouts.md
    └── checklist.md
```

### SKILL.md 结构

```markdown
---
name: <技能名称>
description: <简短描述>
triggers:
  - <触发词 1>
  - <触发词 2>
od:
  mode: prototype        # prototype | deck
  platform: desktop     # desktop | mobile
  scenario: design      # design | marketing | operation | ...
  default_for: <模式>  # 可选：此技能作为某模式的默认
  featured: true       # 可选：显示为特色技能
  fidelity: high        # 可选：high | medium | low
---

# 技能说明

<技能的详细说明和工作流程>
```

### od 前置 Matter 扩展字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `od.mode` | string | 技能模式：prototype 或 deck |
| `od.platform` | string | 平台：desktop 或 mobile |
| `od.scenario` | string | 场景分类 |
| `od.design_system.requires` | string[] | 必需的设计系统 |
| `od.default_for` | string | 作为默认技能的模式 |
| `od.featured` | boolean | 显示为特色技能 |
| `od.fidelity` | string | 保真度：high、medium、low |
| `od.speaker_notes` | boolean | 是否生成演讲者备注 |
| `od.animations` | string[] | 动画效果列表 |
| `od.example_prompt` | string | 示例提示词 |

## 添加自定义技能

### 步骤

1. 在 `skills/` 目录下创建新文件夹
2. 添加 `SKILL.md` 文件
3. 重启守护进程
4. 新技能出现在选择器中

### 示例：创建简单的落地页技能

```markdown
---
name: startup-landing
description: 为初创公司创建极简落地页
triggers:
  - startup landing page
  - 初创公司落地页
od:
  mode: prototype
  platform: desktop
  scenario: marketing
  featured: true
  fidelity: high
---

# Startup Landing Skill

此技能生成初创公司极简风格落地页。

## 工作流程

1. 收集基本信息：公司名称、产品描述、核心价值主张
2. 选择设计方向
3. 生成单页 HTML

## 输出规范

- 单页 HTML
- 使用 CSS Grid 和 Flexbox
- 响应式设计
- 加载时间 < 2s
```

## 视觉方向

当用户没有品牌时，代理发出 5 种策划的视觉方向，每种都有确定的 OKLch 调色板和字体栈：

### 1. Editorial Monocle（编辑单片镜）

经典编辑风格，严谨排版，杂志质感。

### 2. Modern Minimal（现代极简）

干净的线条，大留白，极简美学。

### 3. Warm Soft（温暖柔和）

柔和的边缘，温暖的色调，平易近人。

### 4. Tech Utility（技术实用）

功能导向，技术美学，数据可视化优先。

### 5. Brutalist Experimental（粗野主义实验）

大胆对比，打破常规，反设计美学。
