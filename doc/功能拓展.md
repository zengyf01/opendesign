# 功能拓展

## 添加自定义技能

### 基础结构

在 `skills/` 目录下创建新文件夹：

```
skills/my-custom-skill/
├── SKILL.md           # 必填
├── assets/            # 可选
│   └── template.html
└── references/        # 可选
    ├── themes.md
    └── checklist.md
```

### SKILL.md 模板

```markdown
---
name: my-custom-skill
description: 简短描述（显示在选择器中）
triggers:
  - 触发词1
  - 触发词2
od:
  mode: prototype        # prototype | deck
  platform: desktop      # desktop | mobile
  scenario: design       # design | marketing | operation | engineering | product | finance | hr | sale | personal
  featured: false        # 是否显示为特色
  fidelity: high         # high | medium | low
---

# 技能名称

详细说明...

## 工作流程

1. 步骤一
2. 步骤二

## 输出规范

- 规范一
- 规范二
```

### 重启守护进程

```bash
# 停止当前运行
pnpm tools-dev stop

# 重启
pnpm tools-dev run web
```

新技能出现在技能选择器中。

## 导入 Claude Design ZIP

Open Design 支持导入 Anthropic Claude Design 导出的 ZIP 文件：

1. 下载 Claude Design 项目为 ZIP
2. 在欢迎对话框中，将 ZIP 拖放到对话框
3. 守护进程解析 ZIP 到项目目录
4. 代理可以继续编辑

导入端点：`POST /api/import/claude-design`

## 自定义媒体生成器

### 配置媒体 API

在 Settings → Media Generation 中配置：

- **图像:** gpt-image-2 (Azure/OpenAI)
- **视频:** Seedance 2.0 (ByteDance)、HyperFrames (HeyGen)
- **音频:** Suno v5、Udio v2、Lyria 2、MiniMax TTS

### 环境变量

```bash
# 媒体配置目录（可选）
OD_MEDIA_CONFIG_DIR=/path/to/media-config

# 数据目录（覆盖所有数据）
OD_DATA_DIR=/path/to/data
```

### 调用媒体生成器

Agent 通过守护进程注入的 `OD_*` 环境变量调用媒体生成器：

```bash
echo "OD_BIN=$OD_BIN"
echo "OD_PROJECT_ID=$OD_PROJECT_ID"
echo "OD_PROJECT_DIR=$OD_PROJECT_DIR"
echo "OD_DAEMON_URL=$OD_DAEMON_URL"
```

## Sidecar IPC 扩展

### IPC 协议

桌面应用通过 sidecar IPC 与守护进程通信：

**Stamp 字段:**
- `app` — 应用标识
- `mode` — 运行模式
- `namespace` — 命名空间
- `ipc` — IPC 通道
- `source` — 源标识

**IPC 命令:**
- `STATUS` — 获取运行时状态
- `EVAL` — 执行 JavaScript
- `SCREENSHOT` — 截图
- `CONSOLE` — 获取控制台日志
- `CLICK` — 模拟点击
- `SHUTDOWN` — 关闭

### 添加新命令

在 `packages/sidecar-proto/` 中定义新命令：

```typescript
export enum SidecarCommand {
  // ... 现有命令
  MY_COMMAND = 'MY_COMMAND',
}
```

## 数据迁移

### 迁移到桌面应用

当从源码开发迁移到打包桌面应用时，数据目录不同：

| 平台 | 桌面应用数据目录 |
|---|---|
| macOS | `~/Library/Application Support/Open Design/namespaces/release-stable/data` |
| Windows | `%APPDATA%\Open Design\namespaces\release-stable-win\data` |
| Linux | `~/.config/Open Design/namespaces/release-stable-linux/data` |

### 使用 OD_LEGACY_DATA_DIR 迁移

```bash
# 设置旧数据目录环境变量
OD_LEGACY_DATA_DIR=/path/to/old/repo/.od /Applications/Open\ Design.app/Contents/MacOS/Open\ Design
```

迁移后，守护进程记录 `[od-migrate] migration complete`。

## MCP 服务器

Open Design 可作为 MCP 服务器运行，允许 Agent 通过 stdio 访问设计文件：

### 配置

在 Claude Code 或其他 MCP 客户端中配置：

```json
{
  "mcpServers": {
    "open-design": {
      "command": "node",
      "args": ["path/to/apps/daemon/dist/cli.js", "mcp"]
    }
  }
}
```

### MCP 工具

- `design_project_list` — 列出项目
- `design_project_read` — 读取项目文件
- `design_project_write` — 写入项目文件
