# 智能代理

## 支持的 CLI

Open Design 自动检测 PATH 上安装的编码智能体 CLI。目前支持 **16 种 CLI**：

| CLI | 命令 | 协议 | 特点 |
|---|---|---|---|
| Claude Code | `claude` | claude-stream-json | 默认优先 |
| Codex CLI | `codex` | json-event-stream | |
| Devin for Terminal | `devin` | acp-json-rpc | ACP 协议 |
| Cursor Agent | `cursor-agent` | json-event-stream | |
| Gemini CLI | `gemini` | json-event-stream | |
| OpenCode | `opencode` | json-event-stream | |
| Qwen Code | `qwen` | plain | |
| Qoder CLI | `qodercli` | qoder-stream-json | |
| GitHub Copilot CLI | `copilot` | copilot-stream-json | |
| Hermes | `hermes` | acp-json-rpc | ACP 协议 |
| Kimi CLI | `kimi` | acp-json-rpc | ACP 协议 |
| Pi | `pi` | pi-rpc | stdio JSON-RPC |
| Kiro CLI | `kiro-cli` | acp-json-rpc | ACP 协议 |
| Kilo | `kilo` | acp-json-rpc | ACP 协议 |
| Mistral Vibe CLI | `vibe` | acp-json-rpc | ACP 协议 |
| DeepSeek TUI | `deepseek` | plain | |

**ACP** = Agent Client Protocol（智能体客户端协议）

## 协议适配器

### 协议类型

| 协议 | CLIs | 说明 |
|---|---|---|
| `claude-stream-json` | Claude Code | Claude Code 专用流式 JSON 格式 |
| `qoder-stream-json` | Qoder CLI | Qoder 专用流式 JSON |
| `copilot-stream-json` | Copilot CLI | Copilot 专用流式 JSON |
| `json-event-stream` | Codex、Gemini、OpenCode、Cursor Agent | 标准 JSON 事件流 |
| `acp-json-rpc` | Devin、Hermes、Kimi、Kiro、Kilo、Mistral Vibe | ACP JSON-RPC |
| `pi-rpc` | Pi | Pi stdio JSON-RPC |
| `plain` | Qwen Code、DeepSeek TUI | 纯文本输出 |

### 适配器实现位置

- `apps/daemon/src/agents.ts` — PATH 扫描 + CLI 适配器定义
- `apps/daemon/src/agents/` — 各协议的专用解析器

## 自动检测

守护进程在启动时扫描 PATH 和常见用户工具链目录，查找上述 CLI。可用 CLI 成为候选设计引擎，显示在模型选择器中。

### 扫描目录

守护进程扫描以下位置：
1. `PATH` 环境变量中的所有目录
2. 常见用户工具链目录（如 `~/.local/bin`、`~/go/bin`）

### 重新扫描

如果安装了新 CLI 但未检测到，可以在 **Settings → Execution & model** 中点击 **Rescan** 重新扫描。

### 手动指定配置目录

如果使用多个 Claude 配置文件，设置 **Settings → Execution & model → Claude Code config directory** 指向特定配置文件路径，如 `~/.claude-2`。

## BYOK 代理（API 模式）

当没有本地 CLI 时，Open Design 支持通过 API 代理连接到云端模型：

### 支持的提供商

| 提供商 | 端点 | 说明 |
|---|---|---|
| Anthropic | `/api/proxy/anthropic/stream` | Claude 系列模型 |
| OpenAI | `/api/proxy/openai/stream` | GPT 系列模型 |
| Azure OpenAI | `/api/proxy/azure/stream` | Azure GPT 系列 |
| Google Gemini | `/api/proxy/google/stream` | Gemini 系列模型 |

### SSRF 防护

BYOK 代理在守护进程边缘阻止内部 IP 访问：

- **允许:** loopback 地址（如 `127.0.0.1`、`localhost`）用于本地 LLM 提供商（Ollama、LM Studio）
- **拒绝:** 非 loopback 私有地址、link-local、CGNAT、多播、保留地址、跳转目标
- **禁用上游重定向:** 守护进程边缘不跟随上游重定向

### 配置 API 密钥

1. 打开 Settings
2. 选择 Execution & model
3. 选择提供商（Anthropic / OpenAI / Azure / Google Gemini）
4. 输入 Base URL 和 API Key
5. 选择模型

## 本地 CLI vs API 模式

| 特性 | 本地 CLI | API 模式 |
|---|---|---|
| 需要本地安装 | 是 | 否 |
| 使用本地工具（Read/Write/Bash） | 是 | 否 |
| Skills 支持 | 是 | 部分 |
| 文件系统产物 | 是 | 否（IndexedDB） |
| PPTX 导出 | 是 | 否 |
| 延迟 | 低 | 取决于网络 |

两种模式共享相同的 `<artifact>` 解析器和沙箱 iframe。区别仅在于传输方式和系统提示词传递（本地 CLI 将组合提示词折叠到用户消息中）。

## 环境变量注入

守护进程在生成 Agent 时注入以下环境变量：

| 变量 | 说明 |
|---|---|
| `OD_BIN` | 守护进程 CLI 的绝对路径（`apps/daemon/dist/cli.js`） |
| `OD_DAEMON_URL` | 运行中守护进程的 URL |
| `OD_PROJECT_ID` | 活跃项目的 ID |
| `OD_PROJECT_DIR` | 活跃项目的文件目录 |

这些变量使 Agent 能够通过本地 `od` CLI 调用媒体生成器。

## 代理故障排除

### "no agents found on PATH"

安装至少一个支持的 CLI，或在 Settings 中切换到 API 模式并配置提供商密钥。

### Claude Code 认证失败

```bash
# 检查认证状态
claude auth status --text

# 如果需要，登录
claude
/login
exit

# 测试非交互模式
printf 'hello' | claude -p --output-format stream-json --verbose --permission-mode bypassPermissions
```

### Codex 加载过多插件上下文

启动 Open Design 时设置 `OD_CODEX_DISABLE_PLUGINS=1`：

```bash
OD_CODEX_DISABLE_PLUGINS=1 pnpm tools-dev
```

### 守护进程 /api/chat 返回 500

通常是 CLI 拒绝了参数。不同 CLI 有不同的 argv 形状。检查守护进程终端的 stderr 尾部，或在 `apps/daemon/src/agents.ts` 的 `buildArgs` 中调整。

## 代理元数据

### 模型能力检测

每种代理有不同的能力（如多模态、工具调用、流式输出）。守护进程通过以下方式检测：

1. **PATH 扫描** — 在 `PATH` 中查找可执行文件
2. **版本查询** — 运行 `<cli> --version` 获取版本信息
3. **能力探测** — 尝试以有限参数启动以探测支持的功能

### 代理选择

用户在 Settings 中选择活跃代理。选择后，守护进程：
1. 验证 CLI 在 PATH 上
2. 建立适配器连接
3. 持续监控进程状态
4. 处理 stdout/stderr 流
